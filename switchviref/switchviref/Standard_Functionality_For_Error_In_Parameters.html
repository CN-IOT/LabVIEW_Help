使用标准功能处理参数错误

许多 LabVIEW 节点（例如 VI）包含可用于管理错误的参数中的错误。这些参数通常提供相同的标准功能。当一个节点表现出不同的参数功能时，例外情况会记录在该节点的参考资料中。行为标准错误如下

注意  某些节点，例如错误处理VI，包含参数错误，该错误不提供标准功能错误，但包含标准群集错误。
error in描述在此节点运行之前发生的错误情况。默认为无错误。如果在此节点运行之前发生错误，则节点将错误值传递给错误输出。只有在该节点运行之前没有发生错误时，该节点才能正常运行。如果此节点运行时发生错误，它会正常运行并在error out中设置自己的错误状态。

使用错误输入和错误输出来检查错误并通过将错误从一个节点连接到下一个节点的错误来指定执行顺序。

集群中的错误包含以下集群元素：

如果在此节点运行之前发生错误，则状态为 TRUE (X)，或者为 FALSE（复选标记）以指示警告或在此节点运行之前未发生错误。默认值为假。
code是错误或警告代码。默认值为 0。如果状态为 TRUE，则代码为错误代码。如果状态为 FALSE，则代码为 0 或警告代码。
source指定错误或警告的来源，在大多数情况下，它是产生错误或警告的节点的名称。默认值为空字符串。
