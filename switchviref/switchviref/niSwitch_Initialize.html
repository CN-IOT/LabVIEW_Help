niSwitch 初始化 VI

返回会话句柄，用于识别所有后续 NI-SWITCH 调用中的开关模块。

该VI为资源名称参数中指定的开关模块创建一个新的IVI仪器驱动会话。如果多个拓扑对该开关模块有效，NI-SWITCH 将使用 MAX 中指定的默认拓扑。

默认情况下，开关模块复位为已知状态。

如果与指定资源的会话存在于另一个进程中，则返回错误。如果在同一个进程中为同一个拓扑的同一个资源调用该VI两次，则返回同一个会话。

id 查询被忽略。

因为 NI-SWITCH 支持多个开关模块，它总是查询开关模块以确定安装了哪个模块。因此，即使该参数设置为 VI FALSE，该 VI 也可能返回 ID 查询失败错误。

reset device指定初始化时是否复位开关模块。默认值为 TRUE (1)。

注意重新启动计算机后  第一次调用niSwitch Initialize With Topology VI将重置硬件。这是不支持重置标志的唯一情况。

设备将重置。
错误 (0)

设备不会重置。

error in描述在此节点运行之前发生的错误情况。该输入提供了功能上的标准错误。

如果在该 VI 运行之前发生错误，则状态为 TRUE (X)，或者为 FALSE（复选标记）以指示警告或在该 VI 运行之前未发生错误。默认值为假。

code是错误或警告代码。默认值为 0。如果状态为 TRUE，则代码为非零错误代码。如果状态为 FALSE，则代码为 0 或警告代码。

source描述了错误或警告的来源，在大多数情况下，它是产生错误或警告的 VI 的名称。默认值为空字符串。

instrument handle out identifies a particular NI-SWITCH session established with the niSwitch Initialize With Topology VI, the niSwitch Initialize With Options VI, or the niSwitch Initialize VI and used for all subsequent NI-SWITCH calls.

error out contains error information. This output provides standard error out functionality.

status is TRUE (X) if an error occurred or FALSE (checkmark) to indicate a warning or that no error occurred.

code是错误或警告代码。如果状态为 TRUE， 则代码为非零错误代码。如果状态为 FALSE， 则代码为 0 或警告代码。

source描述了错误或警告的来源，在大多数情况下，它是产生错误或警告的 VI 的名称。
