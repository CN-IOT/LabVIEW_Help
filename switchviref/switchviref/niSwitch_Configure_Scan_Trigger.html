niSwitch 配置扫描触发 VI

为使用niSwitch 配置扫描列表 VI建立的扫描列表配置扫描触发器。

请参阅设备确定交换模块是否支持扫描。

该VI设置开关预期接收输入触发的位置，以在扫描列表中前进。该VI还设置完成扫描列表中的条目后生成扫描高级信号的位置。

细节

仪器句柄标识通过niSwitch Initialize With Topology VI、niSwitch Initialize With Options VI或niSwitch Initialize VI建立的特定 NI-SWITCH 会话。

扫描延迟指定您希望交换机设备在创建路径后等待的最短时间，直到它在扫描高级输出通道上断言触发。NI-SWITCH 使用该值来设置扫描延迟属性。扫描延迟是建立时间之外的。

默认值：0.0（秒）

触发输入指定您希望开关模块在扫描期间使用的触发源。NI-SWITCH 使用该值来设置触发输入属性。默认值为外部。

切换模块在扫描列表中遇到分号时，在指定位置等待触发。当触发发生时，开关设备前进到扫描列表中的下一个条目，并等待来自trigger input中指定位置的触发。

立即 (0)

立即处理扫描列表中的下一个条目。
外部 (1)

在处理扫描列表中的下一个条目之前等待前连接器上的触发器。与前连接器相同。
软件触发功能 (2)

等待直到调用 niSwitch Send Software Trigger VI。

TTLx  (3)

在处理扫描列表中的下一个条目之前，等待 PXI 或 SCXI 触发线x上的触发。
后连接器 (4)

仅对 SCXI 有效。在处理扫描列表中的下一个条目之前，等待后连接器上的触发器。
前连接器 (5)

在处理扫描列表中的下一个条目之前等待前连接器上的触发器。与外部相同。
模块后连接器 <1..12>  (6)

仅对 SCXI 有效。在处理扫描列表中的下一个条目之前，等待插槽 <1..12> 的后连接器上的触发器。可用作调用niSwitch Route Trigger Input VI的替代方法。
模块前连接器 <1..12>  (7)

仅对 SCXI 有效。在处理扫描列表中的下一个条目之前，等待插槽 <1..12> 的前连接器上的触发器。可用作调用niSwitch Route Trigger Input VI的替代方法。

高级扫描输出（外部）指定高级扫描输出触发信号的输出端。NI-SWITCH 使用该值来设置扫描高级输出属性。默认值为外部。

交换机处理完扫描列表中的每个条目后，它会等待扫描延迟中指定的时间长度，然后在扫描高级输出中指定的行上触发触发器 。

无 (1)

不产生扫描高级输出触发。
外部 (2)

将扫描高级输出触发发送到开关模块的前连接器。与前连接器相同。
TTLx  (3)

将扫描高级输出触发发送到 PXI 或 SCXI 触发线x。
PXI 星 (4)

在处理扫描列表中的下一个条目之前，交换机会一直等待，直到它在 PXI 星形触发总线上接收到一个触发信号。
后连接器 (5)

仅对 SCXI 有效。将扫描高级输出触发发送到开关模块的后连接器。
前连接器 (6)

将扫描高级输出触发发送到开关模块的前连接器。与外部相同。
后连接器 <1..12>  (7)

仅对 SCXI 有效。将扫描高级输出触发器发送到插槽 <1..12> 的后连接器。可用作调用niSwitch Route Scan Advanced Output VI的替代方法。
前连接器 <1..12>  (8)

仅对 SCXI 有效。将扫描高级输出触发发送到插槽 <1..12> 的前连接器。可用作调用niSwitch Route Scan Advanced Output VI的替代方法。

error in描述在此节点运行之前发生的错误情况。该输入提供了功能上的标准错误。

如果在该 VI 运行之前发生错误，则状态为 TRUE (X)，或者为 FALSE（复选标记）以指示警告或在该 VI 运行之前未发生错误。默认值为假。

code是错误或警告代码。默认值为 0。如果状态为 TRUE，则代码为非零错误代码。如果状态为 FALSE，则代码为 0 或警告代码。

source描述了错误或警告的来源，在大多数情况下，它是产生错误或警告的 VI 的名称。默认值为空字符串。

instrument handle out identifies a particular NI-SWITCH session established with the niSwitch Initialize With Topology VI, the niSwitch Initialize With Options VI, or the niSwitch Initialize VI and used for all subsequent NI-SWITCH calls.

error out contains error information. This output provides standard error out functionality.

status is TRUE (X) if an error occurred or FALSE (checkmark) to indicate a warning or that no error occurred.

code is the error or warning code. If status is TRUE, code is a non-zero error code. If status is FALSE, code is 0 or a warning code.

source describes the origin of the error or warning and is, in most cases, the name of the VI that produced the error or warning.

Details

For synchronous scanning, set scan advanced output to None and define trigger input. For handshaking, trigger input and scan advanced output must be set.

Triggers generally originate from the SCXI or PXI trigger bus, the terminals on the front connector of the switch module, or the AUX connector on the rear of the switch module (referred to as the rear connector and supported by SCXI only).

When scanning a module that is not directly cabled to the trigger from its own front or rear connector, use the niSwitch Route Trigger Input VI and/or the niSwitch Route Scan Advanced Output VI to route the triggers on the cabled module to one of the backplane lines. When using these VIs to route triggers to the backplane, specify the backplane trigger location when calling the niSwitch Configure Scan Trigger VI (even if it is the module cabled to the trigger source).
