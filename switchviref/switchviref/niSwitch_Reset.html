niSwitch 重置 VI

断开所有创建的路径并将开关模块返回到初始化状态。配置通道和源通道设置保持不变。

如果错误描述了调用该VI之前发生的错误，该VI仍会尝试重置开关。但是，如果尝试失败，该VI将返回从error in参数传入的错误信息。

仪器句柄标识通过niSwitch Initialize With Topology VI、niSwitch Initialize With Options VI或niSwitch Initialize VI建立的特定 NI-SWITCH 会话。

error in描述在此节点运行之前发生的错误情况。除了以下例外，此输入在功能上提供标准错误。即使之前发生错误，此节点也会正常运行。如果在该节点运行之前发生错误，则该节点将错误值传递给错误输出，并且不会记录该节点运行时发生的错误。如果在该节点运行时发生错误并且之前没有发生错误，则该节点将自己的错误状态设置为错误输出。默认为无错误。

如果在该 VI 运行之前发生错误，则状态为 TRUE (X)，或者为 FALSE（复选标记）以指示警告或在该 VI 运行之前未发生错误。默认值为假。

code是错误或警告代码。默认值为 0。如果状态为 TRUE，则代码为非零错误代码。如果状态为 FALSE，则代码为 0 或警告代码。

source描述了错误或警告的来源，在大多数情况下，它是产生错误或警告的 VI 的名称。默认值为空字符串。

instrument handle out标识一个特定的 NI-SWITCH 会话，该会话通过niSwitch Initialize With Topology VI、niSwitch Initialize With Options VI或niSwitch Initialize VI 建立，并用于所有后续的 NI-SWITCH 调用。

error out包含错误信息。此输出提供标准错误输出功能。

如果发生错误，则状态为 TRUE (X)，或者为 FALSE（复选标记）以指示警告或未发生错误。

code是错误或警告代码。如果状态为 TRUE， 则代码为非零错误代码。如果状态为 FALSE， 则代码为 0 或警告代码。

source描述了错误或警告的来源，在大多数情况下，它是产生错误或警告的 VI 的名称。
