在 IVI 中使用 NI 开关

创建 IVI 逻辑名称和相应的驱动程序会话以在 IVI 中使用 National Instruments 交换机模块。完成以下步骤以创建 IVI 逻辑和相应的驱动程序会话。

  1. 打开测量和自动化资源管理器 (MAX)。
  2. 在 MAX 配置树中展开IVI Drivers 。
  3. 创建一个新的驱动程序会话。
    1. 右键单击配置树中的Driver Sessions ，然后选择Create New (case-sensitive)。配置面板在配置窗口中打开。
    2. 完成以下步骤以配置新的驱动程序会话。
      1. 选择Software选项卡，然后从 Software Module 列表框中选择niSwitch 。
      2. 选择硬件选项卡，然后从硬件资产列表框中选择要使用的交换机模块。
        注意  要创建新的硬件资产，请单击硬件资产列表框右侧的新建。
      3. 选择General选项卡，然后从 Simulate With 列表框中选择Don't Simulate 。
      4. 在 Driver Setup 文本框中，输入要使用的交换机模块的驱动程序设置字符串。有关驱动程序设置字符串的详细信息，请参阅确定驱动程序设置字符串。
  4. 创建一个新的逻辑名称。
    1. 右键单击配置树中的逻辑名称，然后选择新建（区分大小写）。配置面板在配置窗口中打开。
    2. 要配置新的逻辑名称，请从Driver Session列表框中选择您在步骤 3 中创建的驱动程序会话。
      注意  要查看您选择的驱动程序会话的属性，请单击驱动程序会话列表框右侧的转到。
  5. 要保存 IVI 配置，请单击配置视图顶部的保存 IVI 配置。如果您决定不保存更改，请单击恢复以恢复到以前的配置。

确定驱动程序设置字符串

完成以下步骤以确定交换机模块的驱动程序设置字符串。

  1. 确定拓扑名称。
    1. 打开 NI Switches 帮助文件。
    2. 选择内容选项卡。
    3. 展开Devices手册，然后选择您要使用的开关模块。开关模块的设备概览显示在帮助窗口中。
    4. 在设备概述中，操作模式表列出了每个支持的拓扑的拓扑和软件名称。拓扑名称是表中列出的第一个软件名称，而不是括号中的常量名称。例如，如果您想在 48x1 Mux 模式下使用 NI PXI-2501，拓扑名称为“2501/1-Wire 48x1 Mux”而不是“ NISWITCH_TOPOLOGY_2501_1_WIRE_48X1_MUX ”。
  2. 使用拓扑名称，确定驱动程序设置字符串。
    • NI-DAQmx 设备— 驱动设置字符串是topology: <topology name>。例如，如果您在 48x1 Mux 模式下使用 NI PXI-2501，驱动设置字符串为topology:2501/1-Wire 48x1 Mux。
