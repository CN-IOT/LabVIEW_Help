连接和断开列表语法

您可以将连接列表和断开列表与niSwitch Connect Channels (Multiple)和niSwitch Disconnect Channels (Multiple)一起使用来创建一个或多个切换操作的字符串。对于具有多个操作的列表，逗号分隔每个操作。NI-SWITCH 验证连接和断开列表，如果返回错误则中止列表的执行。

将通道设置为配置通道，以允许 NI-SWITCH 使用该通道创建内部路径。如果两个通道之间的路径包括一个或多个通道，则中间通道应设置为配置通道。如果存在多个中间通道，并且通道类型设置为配置通道，NI-SWITCH 会选择使用中间通道。

注意  配置通道在其他交换机 API 中也可能被称为“保留用于路由”。

切换操作可以以两种方式之一连接和断开通道。请参考下表确定切换操作。

切换操作 句法 描述
指定端点 频道 1 -> 频道 2
 NI-SWITCH 搜索端点之间的可用路径以连接或断开连接。指定的两个通道必须在同一设备上，并且它们之间必须存在可用路径。
指定显式路径 [频道1->频道2->频道3]
 您定义路径的端点和任何中间通道。*
*操作顺序不保证执行顺序。要确保特定顺序，请使用多个连接和断开连接列表。

提示  NI-SWITCH 忽略空白和换行。使用这些来格式化冗长列表的外观。

连接和断开列表示例

示例 1

如果要使用指定端点和显式路径的切换操作创建连接或断开列表，请用逗号分隔切换操作。

语法：Operation1、Operation2、Operation3

示例：c0 -> r0, [c1 -> r2 -> c5], r0 -> c4

示例 2

如果channel0、com0和AB0之间存在路径，则可以通过将com0的通道类型设置为配置通道，将channel0连接到AB0。

语法：通道 1 -> 通道 2

示例：通道 0 -> AB0

示例 3

如果要连接矩阵的第 1 列和第 5 列，并使用第 2 行完成连接，则必须在连接前将第 2 行的通道类型设置为配置通道。

语法：[channel1 -> channel2 -> channel3]

示例：[c1 -> r2 -> c5]
